# chetor: دستورالعمل گام‌به‌گام

1. اگر مراحل به UI یا قوانین روز وابسته است (پنل سرویس‌ها، کارهای اداری)، اول وب را چک کن که مراحل امروز درست باشد. از حافظهٔ کهنه دستورالعمل نده.
2. خروجی: کارت فارسی. فقط گام‌های خود کاربر، شماره‌دار. هر گام یک جملهٔ سادهٔ 15 تا 20 کلمه‌ای. پیش‌نیازها اول. لینک و مسیر داخل code. هیچ گامی که «برای من بدیهی و برای کاربر گنگ» است جا نماند.
3. اگر ادامهٔ کار را خودم می‌توانم انجام دهم (مثل ذخیرهٔ توکن در جای درست و تستش با یک کال واقعی)، در پایان پیشنهاد بده؛ بدون تأیید انجام نده.
4. طول: کوتاه‌ترین نسخهٔ کامل. اگر مسیر چند حالت دارد، اول حالت اصلی، بعد یک خط برای استثناها.
