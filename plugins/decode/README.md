# decode

Code in product language: explain any file, codebase, or diff to a PM who doesn't read code. Persian-first.

کد به زبان محصول، برای مدیر محصولی که کد نمی‌خونه ولی مهندسی نرم‌افزار رو می‌فهمه.

## سه حالت

- `/decode src/checkout.ts`: شناسنامهٔ فایل توی چت: نقش، تریگر، ورودی و خروجی، دست‌درازی، ریسک محصولی، مدرک (لنگر خط).
- `/decode src` یا `/decode atlas`: اطلس HTML مستقل از کل کدبیس: نقشهٔ ماژول‌ها، سفرهای کاربر، شناسنامهٔ فایل‌ها، با جستجو و تم روشن و تاریک. خروجی: `atlas.html` در ریشهٔ ریپو.
- `/decode diff` یا `/decode PR 42`: ترجمهٔ تغییرات به زبان محصول، مثل ریلیز نوت، با فهرست «چی رو تست کنیم».

## قانون

کلمات معماری آزاد (API، دیتابیس، صف). کلمات سینتکس ممنوع. روایت خط‌به‌خط ممنوع. هر ادعا با لنگر `file:line` تا قابل ممیزی باشه؛ رفتار انتظاری از روی تست‌ها، نه حدس.
